# Cancer Cell Metastasis 3D Viewer

This folder contains an interactive 3D OpenGL viewer for visualizing cancer cell tumor simulations. Cells are loaded from a CSV snapshot produced by the agent-based tumour-growth model and rendered as colour-coded spherical nodes in a navigable 3D environment.

## Overview

This sub-project provides a real-time 3D visualization tool for tumor snapshots generated by an agent-based cancer simulation. Each cell in the simulation is exported to a CSV file with its 3D lattice coordinates and biological phenotype. The viewer loads this data and renders every cell as a bumpy sphere, coloured by phenotype, inside an interactive OpenGL scene.

Key features:
- **Phenotype-aware colouring** — necrotic core, surface, condensing, and non-condensing cells each have a distinct colour.
- **Hierarchical scene graph** — cells are modelled as `HierarchicalNode` objects composed of a central sphere and smaller bump spheres.
- **Trackball navigation** — quaternion-based rotation lets you freely orbit the tumour.
- **Ray-cast picking** — click any cell to select it; selected objects emit a highlight glow.
- **AABB collision** — fast axis-aligned bounding-box intersection for picking and spatial queries.

## Phenotype Color Legend

| Phenotype | Colour | Hex | Biological Meaning |
|---|---|---|---|
| `necrotic` | ⚫ Black | `#000000` | Dead cells in the hypoxic core |
| `surface` | 🟣 Magenta | `#FF00FF` | Outermost cells with empty neighbours |
| `condensing` | 🔵 Blue | `#0000FF` | Cells with a condensation phenotype |
| `non-condensing` | 🔴 Red | `#FF0000` | Cells with a non-condensation phenotype |

Bump spheres on each cell body are rendered in a warm peach tone (`rgb(1.0, 0.698, 0.4)`) regardless of phenotype.

## Data Format

Both CSV files are generated by the agent-based tumour simulation. Their schemas are described below.

### tumor\_cells.csv

One row per exported cell. Coordinates are relative to the lattice centre (i.e., `position = lattice_index - L/2`) for simplicity at the 3D Viewer setup.

| Column | Type | Description |
|---|---|---|
| `x` | int | X coordinate (centred) |
| `y` | int | Y coordinate (centred) |
| `z` | int | Z coordinate (centred) |
| `phenotype` | str | One of `necrotic`, `surface`, `condensing`, `non-condensing` |
| `oxygen` | float | Local oxygen concentration at cell position |
| `C` | float | Hypoxia ratio: `1 - oxygen / O_MAX` |
| `b` | float | Cell division probability at this simulation step |
| `d` | float | Cell death probability at this simulation step |
| `sim_time` | int | Simulation time step of the snapshot |

> **Downsampling note:** The exporter uses a stratified surface-preserving strategy. Surface cells and necrotic cells are always exported in full. Interior living cells are randomly sampled to keep the total at or below `max_cells` (default 5 000). This preserves the two biologically critical structures — the tumour boundary and the necrotic core — while keeping rendering performance manageable.

### tumor\_history.csv

One row per simulation time step. Used to plot tumour growth dynamics.

| Column | Type | Description |
|---|---|---|
| `sim_time` | int | Simulation step index (1-based) |
| `population` | int | Total number of living cells |
| `metastatic_cells` | int | Number of cells that have undergone metastatic transition |
| `avg_b` | float | Mean division probability across all cells |
| `avg_d` | float | Mean death probability across all cells |
| `avg_C` | float | Mean hypoxia ratio across all cells |
| `R_ratio` | float | Necrotic-core radius / total-tumour radius |

## Architecture

### Module Descriptions

| Module | Responsibility |
|---|---|
| `viewer.py` | Creates the GLUT window, owns the OpenGL context, drives the render loop, and delegates picking/movement/placement to the `Scene`. Also builds the initial scene from the CSV. |
| `scene.py` | Maintains the `node_list` and `selected_node`. Implements scene-level operations: `render`, `pick`, `place`, `move_selected`, `scale_selected`, `rotate_selected_color`. |
| `node.py` | Defines the node class hierarchy (`Node → Primitive → Sphere / Cube`, `Node → HierarchicalNode`), the `AABB` ray-intersection helper, and the `init_primitives()` function that compiles OpenGL display lists. |
| `cancer_cell.py` | Subclasses `HierarchicalNode` to build a cell from one central `Sphere` and five randomly-placed bump `Sphere` objects. Colours the body by phenotype. |
| `interaction.py` | Registers GLUT callbacks (`glutMouseFunc`, `glutMotionFunc`, `glutKeyboardFunc`) and translates raw events into high-level viewer commands via a callback registry. |
| `trackball.py` | Implements a virtual trackball using quaternion arithmetic. `drag_to(dx, dy)` accumulates a rotation quaternion and exposes a `4×4 matrix` consumed by the render loop. |

## Installation & Usage

### Python Version

- Python 3.8 or higher recommended
- Check your version: `python --version` or `python3 --version`

### Required Libraries

Most libraries are built-in, but you'll need to install:
- Libraries: `numpy`, `pandas` and `PyOpenGL`

Install system GLUT if it is not already present:

```bash
# Debian / Ubuntu
sudo apt-get install freeglut3-dev

# macOS (Homebrew)
brew install freeglut

# Windows
# Download freeglut binaries from https://freeglut.sourceforge.net/
# and place glut32.dll / glut64.dll on your PATH.
```

#### Setup

1. Clone repository:
```bash
git clone https://github.com/b-salgado13/Cancer_Metastasis_Simulations.git
cd Cancer_Metastasis_Simulations\3D_Viewer
```
2. Install dependencies:
```bash
pip install -r requirements.txt
```
3. Place your simulation data:
```bash
mkdir -p data
cp /path/to/tumor_cells.csv   data/
cp /path/to/tumor_history.csv data/
```

### Usage

```bash
python viewer.py
```

The viewer reads `data/tumor_cells.csv` at startup. If the file is not found it prints an error and exits. The window title is **Cancer Cell Modeller**.

To visualise a different snapshot, replace `data/tumor_cells.csv` with any file that conforms to the schema above (the `phenotype` column must contain one of the four recognised values).

## Controls

| Input | Action |
|---|---|
| **Right-click drag** | Orbit / rotate the tumour (trackball) |
| **Middle-click drag** | Pan the camera (translate X/Y) |
| **Scroll wheel up** | Zoom in (translate along Z) |
| **Scroll wheel down** | Zoom out (translate along Z) |
| **Left-click** | Pick / select a cell (highlights it with emissive glow) |

> Keyboard controls are registered but currently inactive. Extend `Interaction.handle_keystroke` to add hotkeys (e.g., reset view, toggle phenotype visibility).

## How It Works

### Rendering Pipeline

Each frame follows this sequence:

1. **`init_view`** — Sets up a perspective projection (`FOV 70°`, near `0.1`, far `1000`) and translates the camera `15` units back along Z.
2. **Trackball matrix** — The current quaternion rotation matrix is multiplied onto the modelview stack via `glMultMatrixf`.
3. **`scene.render()`** — Iterates `node_list`; each `Node.render()` pushes its translation and scaling matrices, sets the colour (or emissive highlight if selected), and calls `render_self()`.
4. **Display lists** — Sphere and cube geometry is compiled once at startup into OpenGL display lists (`G_OBJ_SPHERE`, `G_OBJ_CUBE`, `G_OBJ_PLANE`) and replayed each frame with `glCallList` for maximum throughput.
5. **Grid plane** — After scene objects, the reference grid is drawn with lighting disabled.

### Trackball Rotation

`Trackball.drag_to(x, y, dx, dy)` converts mouse deltas into two quaternions:

- `q_x` — rotation around the **world X axis** by `dy × sensitivity` degrees (pitch).
- `q_y` — rotation around the **world Y axis** by `dx × sensitivity` degrees (yaw).

These are concatenated onto the accumulated rotation quaternion:

```
rotation = q_y ⊗ (q_x ⊗ rotation)
```

The quaternion is normalised every frame to prevent floating-point drift, then converted to a `4×4` rotation matrix via the standard quaternion-to-matrix formula. Sensitivity is fixed at `0.5°` per pixel.

### Ray Casting & Picking

When the user left-clicks, `Viewer.pick(x, y)` is called:

1. **`get_ray`** — Uses `gluUnProject` at `z=0.001` (near plane) and `z=0.999` (far plane) to produce a world-space ray `(start, direction)`.
2. **`scene.pick`** — Iterates every node and calls `node.pick(start, direction, modelView)`.
3. **`AABB.ray_hit`** — Transforms the ray into the node's local space using the inverse of the combined modelview + translation matrix, then applies the slab method to test intersection and return the parametric distance `t`.
4. The node with the **smallest positive `t`** is selected. Selection emits a `[0.3, 0.3, 0.3]` emissive material.

## Simulation Data Pipeline

The full pipeline from simulation to viewer is:

```
"Cancer Metastasis Full python.py"
        │
        ▼
save_cells_csv()          ← stratified downsampling (surface + necrotic always kept)
        │
        ▼
data/tumor_cells.csv      ← x, y, z, phenotype, oxygen, C, b, d, sim_time
        │
        ▼
Viewer.create_sample_scene()
        │
        ▼
CancerCell(phenotype=...).translate(x, y, z)
        │
        ▼
Scene.node_list → OpenGL render loop
```

And for growth statistics:

```
save_history_csv()
        │
        ▼
data/tumor_history.csv    ← sim_time, population, metastatic_cells, avg_b, avg_d, avg_C, R_ratio
```

`tumor_history.csv` is not consumed by the viewer directly but is available for downstream plotting (e.g., with matplotlib or a separate dashboard).
